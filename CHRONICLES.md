## Design Decisions

### Decision 1: Frequency as Primary Interaction Model
Rather than traditional UI buttons, the frequency slider is the primary control. This creates a unique interaction paradigm where users "tune" rather than "click."

**Rationale:** Aligns with project ethos (frequency-based identity, harmonic resonance)  
**Trade-off:** Steeper learning curve, but more memorable UX

### Decision 2: Terminal Aesthetic
Void Terminal design (#00ff41 on #020202) chosen over modern UI frameworks.

**Rationale:** Aligns with "401 bypass" narrative, creates visual distinction  
**Trade-off:** Less familiar to modern web users, but highly recognizable

### Decision 3: In-Memory State Management
Game sessions and mashups stored in-memory rather than database.

**Rationale:** Faster iteration, simpler architecture during development  
**Trade-off:** Data lost on server restart (acceptable for MVP)

### Decision 4: Demucs for Stem Separation
Chosen over Spleeter for superior quality on complex mixes.

**Rationale:** Better audio quality, active development, Meta backing  
**Trade-off:** Slower processing, higher CPU usage

### Decision 5: Smooth Cosine Crossfade
DJ Mixer uses cosine interpolation rather than linear crossfade.

**Rationale:** Perceptually smoother volume transition, more professional sound  
**Trade-off:** Slightly more complex math, but negligible performance impact
